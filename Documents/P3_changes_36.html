Changes

There have definitely been some changes since P2, but the main idea of the website as well as the most basic functionality still remains the same. First of all the navigation structure is the same, which means that we still have the four main pages of the website, with the basic functionality remaining the same.

The main change of the booking site is that there have been added a lot more validation of the users input, than we presented to do in P2. Initially we just planned to have some basic validation as well as a JavaScript-function that confirms the booking to the user.

The changes include:

  • "ticketValid()" is a JavaScript function that we made in order to validate the users input. This means extra validation in addition to the use of the keyword "required" in the html-file. This function checks two things. The first thing is that the movie has to be booked minimum one hour before it is scheduled, so the function takes in the time of the movie the person tries to book (either 5pm, 7pm or 9pm), and checks if there is less than one hour until the movie is scheduled. This function also checks whether the sum of the number of booked tickets equals zero, which we could not accept. Because child tickets are free, there is an extra check for that as well.
  • "bookingConfirmation() works in about the same way as we described it in P2, but it does not make a pop-up window, as we initially described in the mockups in P2. We consider this as a minor change.
  • "updateMovie()" is a JavaScript function that we added for the form to show the image of the movie you're about to book tickets for. We didn't explicitly define this function in P2, but the functionality is shown in the mockups.
  • html_generator is an additional JavaScript file that we added to generate the header and the footer of all pages. This was not a script that we mentioned in P2, but we added this so that we don't have as much duplicate code. This script adds the common html for both the header and the footer when a new page is loaded. In addition to that it also changes the color of the header at the page you are currently at.

Modularity and implementation

Most of the website is unique for each and every site, but there is also some things that are used for all sites, e.g. header and footer. We therefore created a javascript called html_generator which basically just fetches the header and footer elements, and sets the innerHTML of those to the common HTML for the different pages when they're loaded.

Another example of modularity is the use of classes and elements in the CSS . This is only one document for the whole website, so everything written in the CSS that is reused because every HTML-file is affected of the styling in the CSS-file.
